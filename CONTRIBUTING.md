# Contributing to Hidden Treasures Network

Thank you for your interest in contributing to Hidden Treasures Network!

## How to Contribute

### Reporting Bugs

1. Check if the bug has already been reported in the Issues section
2. If not, create a new issue with:
   - Clear title and description
   - Steps to reproduce
   - Expected vs actual behavior
   - Screenshots if applicable

### Suggesting Features

1. Check if the feature has already been suggested
2. Create a new issue with:
   - Clear description of the feature
   - Use cases and benefits
   - Possible implementation approach

### Pull Requests

1. Fork the repository
2. Create a new branch from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. Make your changes
4. Write or update tests as needed
5. Ensure all tests pass
6. Commit your changes with clear messages
7. Push to your fork
8. Submit a pull request

## Development Guidelines

### Code Style

- Use 2 spaces for indentation
- Follow ESLint rules
- Write meaningful variable and function names
- Comment complex logic

### Commit Messages

- Use present tense ("Add feature" not "Added feature")
- Use imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit first line to 72 characters
- Reference issues and pull requests when applicable

### Testing

- Write tests for new features
- Ensure existing tests pass
- Aim for good code coverage

## Project Structure

Please familiarize yourself with the project structure before contributing. See the README.md for details.

## Questions?

Feel free to open an issue for any questions or clarifications needed.
